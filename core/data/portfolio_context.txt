--- Project 4: 8TechBank ---
Type:   Security research + production banking application
Stack:  Flask, Jinja2, PostgreSQL, Docker, JWT

Description:
8TechBank is a dual-mode banking web application built in two parallel versions: (1) a
deliberately vulnerable version exposing 6 common web security flaws for penetration testing
education, and (2) a production-hardened secure version with all vulnerabilities remediated.

Vulnerabilities Covered (Vulnerable Version):
  - SQL Injection (login bypass and data exfiltration)
  - Plaintext password storage
  - Cross-Site Request Forgery (CSRF)
  - Insecure Direct Object Reference (IDOR)
  - Session misconfiguration (persistent, non-expiring sessions)
  - Stored and Reflected Cross-Site Scripting (XSS)
  - Missing HTTP security headers

Security Controls (Hardened Version):
  - Parameterised queries, bcrypt password hashing, CSRF tokens
  - Object-level authorisation checks, secure session management
  - Full security header suite (CSP and more)
  - JWT-based API authentication
  - Dockerized deployment with non-root user
  - OWASP ASVS Level 1 alignment

Demonstrates: Cybersecurity expertise, penetration testing knowledge, secure SDLC practices.
